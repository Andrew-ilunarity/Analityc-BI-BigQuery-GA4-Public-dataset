# 📊 КЕЙС: Наскрізна аналітика та оцінка ефективності маркетингу (GA4 ➡️ BigQuery ➡️ Power BI)

**Стек технологій:** `Google Analytics 4 (GA4)`, `Google BigQuery`, `SQL`, `DBeaver`, `Power BI (DAX, Data Modeling)`.

---

## 📌 Situation (Ситуація)

Компанія накопичує великий масив сирих даних про поведінку користувачів через веб-аналітику **Google Analytics 4**. Проте ці дані існували ізольовано від фінансових звітів та маркетингових витрат. Керівництво не мало єдиного інструменту для оцінки окупності інвестицій в рекламу, через що виникав ризик неефективного розподілу бюджетів між маркетинговими каналами.

## 🎯 Task (Задача)

Побудувати автоматизоване **End-to-End (наскрізне) аналітичне рішення** для об'єднання логів GA4 із даними про рекламні витрати. Головна мета — розрахувати ключові бізнес-метрики (**ROMI, CPO, AOV**) у розрізі маркетингових кампаній, географії та операційних систем, а також візуалізувати їх у вигляді інтерактивного дашборду для топ-менеджменту (CEO / CMO).

## 🛠️ Action (Дії)

Всю розробку рішення я виконав самостійно за **7 годин**, реалізувавши повний цикл обробки даних (ETL):

1. **Екстракція та підготовка даних (SQL & BigQuery):**
* Написав складні модульні SQL-запити в хмарному сховищі **BigQuery** через **DBeaver** для агрегації сирих логів e-commerce подій GA4.
* Використав підхід **CTE (Common Table Expressions)** для створення чистого та оптимізованого конвеєра даних, об'єднавши віртуальні календарі маркетингових витрат із фактами продажів на рівні бази даних.


2. **Моделювання даних та аналітика (Power BI & DAX):**
* Розробив модель даних, успішно вирішивши проблему зв'язку **Many-to-Many (*:*)** між таблицями фактів продажів та витрат за допомогою двонаправленої фільтрації.
* Написав формули на мові **DAX** для розрахунку бізнес-метрик: глобального та локального окупності маркетингу (`ROMI`), вартості замовлення (`CPO`), середнього чека (`AOV`) та кількості унікальних транзакцій.


3. **Проектування та розробка дашборду:**
* Створив багатовкладочний інтерактивний дашборд. На першому плані реалізував головні KPI-картки (`CPO`, `Orders`) для миттєвої оцінки ситуації керівництвом.
* Налаштував **Waterfall chart (діаграму водоспаду)** для наочної демонстрації щоденного накопичення чистого прибутку, а також інтегрував глибоку аналітику трафіку за операційними системами та пристроями.
* Додав детальну зведену матрицю ефективності каналів для швидкого переходу від глобальних трендів до точних цифр.



## 🏆 Result (Результат)

* **Створено єдине джерело правди:** Бізнес отримав готовий інтерактивний інструмент, який за 3 секунди дозволяє оцінити воронку від витрат до окупності. Наприклад, чітко підсвічено безкоштовні канали трафіку (`organic`, `direct`) та зафіксовано здоровий ROMI за платними кампаніями (151% та 249%).
* **Оптимізація швидкості:** Завдяки винесенню агрегації даних на сторону BigQuery та побудові правильної зіркової архітектури, модель даних Power BI працює максимально швидко.

---

### ⚠️ Важливі аналітичні примітки до кейсу (Для співбесіди):

> * **Аналітичне обмеження по географії:** Маркетинговий бюджет у первинних даних фіксувався виключно на рівні кампаній глобально, без прив'язки до локальних ринків. Через це розрахований у звіті локальний ROMI та CPO по окремих країнах є умовним (показники витрат дублюються базою), тоді як глобальні підсумкові значення та розрізи за самими кампаніями є на 100% точними та валідними.
> * **Візуальна цінність:** Основна цінність та архітектурна складність роботи зосереджена саме всередині інтерактивного дашборду Power BI, логіці його зв'язків та DAX-мірок, тому для повної оцінки результату **рекомендовано безпосередньо переглянути сам дашборд у папці `Dashboard`**.
